PDF transcription in OpenCode

Gist link to transcription 'skill'

One of my biggest use-cases for LLMs has been PDF summary and review. For example "did this crystal structure publication report a potency? What assay did they use? Is it commercial or bespoke? Cellular or cell-free?" etc... Sure Claude is great for this - drop in related open-access PDFs and you can summarise a whole field in a few hours. For whatever reason, though, Anthropic's content filter is now too sensitive, and legitimate research into chemistry and drug discovery topics end up killing these review sessions. Constantly.

So it would be nice to have a backup approach. Internally, Claude would convert an academic PDF into text before tokenizing and holding that in its context for downstream summary or review. Remarkably, the best way to do this is not extracting the text and images from with the PDF, but simply taking a snapshot of each page as an image (using, e.g. pdftoppm) and then running them one by one through a vision model to transcribe the text. It sounds like this would be really noisy but, with modern models, for some reason it's not. Today's open weights models can handle multiple columns, text boxes, remove page numbers, and will include figure interpretations.

My own setup uses OpenCode, with whom I have no affiliation. I set up billing as pay-per-token (which they call Zen) to avoid being throttled, and use the model Kimi K2.5, which is a vision model. This approach requires setting up a 'SKILL.md' to instruct the model how to perform the transcription - I posted mine in the gist above. A script to transcribe a bunch of pdfs would thus look like: 


for pdf in ./pdfs/*.pdf; do
    base=$(basename "$pdf" .pdf)
    echo $base
    [ -f "$base.md" ] && continue   # skip if already processed
    mkdir -p "$base"
    pdftoppm -png -r 150 "$pdf" "$base/page"
    opencode run --model opencode/kimi-k2.5 "transcribe the PDF contents in ./$base"
    rm -r $base
done


The transcribed output seems looks good to me and, anecdotally, provides a much more pleasant reading experience compared to a PDF. For example, see martin2014 rendered on sdoc here.

Of course the real utility is in what you do next with the transcribed works. I've found that ingesting several papers as markdown format fits well within the context budget of the open weights models on OpenCode, so a natural next step is to ask for a Q&A document that answers all your questions, pointing to quotes or locations in the source text, for posterity. What's particularly nice about this is that even if LLMs don't improve at all from today, this is still a huge timesaver.
